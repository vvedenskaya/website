# Generative Art с Three.js — пошаговое объяснение

## Общая схема

```
1. Шум (noise)     →  «органичная» случайность
2. Узлы (nodes)    →  позиции кругов и квадратов
3. Связи (lines)   →  линии между узлами
4. Геометрия       →  CircleGeometry, PlaneGeometry, BufferGeometry
5. Материалы       →  MeshBasicMaterial (без света)
6. Анимация        →  requestAnimationFrame + изменение scale/position
```

---

## Часть 1: Шум (Noise)

**Зачем:** Обычный `Math.random()` даёт «шум» — каждая точка независима. В generative art нужна **связность**: соседние точки похожи, далёкие — разные. Так получаются плавные паттерны.

**Что делаем:**
- `noise2D(x, y)` — псевдо-случайное число от 0 до 1, зависящее от координат
- `smoothNoise` — сглаживание между целыми координатами
- `fbm` (fractal Brownian motion) — несколько октав шума, более «природный» вид

**Как использовать:** Передаёшь координаты `(x, y)` — получаешь число. Одинаковые координаты → одинаковый результат (воспроизводимость).

---

## Часть 2: Узлы (Nodes)

**Узел** = точка на плоскости с размером и цветом.

**Алгоритм:**
1. Генерируем много кандидатов (например, 400)
2. Для каждого считаем `density = fbm(nx, ny)` — плотность в этой зоне
3. Если `density < 0.35` — пропускаем (создаём «пустые» зоны)
4. Позицию берём из шума: `x = (noise - 0.5) * bounds`
5. Размер и цвет тоже из шума — чтобы картина была согласованной

**Палитра:** Массив hex-цветов. Индекс цвета = `floor(noise * len) % len`.

---

## Часть 3: Связи (Connections)

**Идея:** Соединяем узлы, которые близко друг к другу.

**Алгоритм:**
1. Для каждой пары узлов `(i, j)` считаем расстояние
2. Если расстояние в нужном диапазоне (не слишком близко, не слишком далеко) — рисуем линию
3. Линия = тонкий прямоугольник (2 треугольника) между точками
4. Толщина и цвет — из шума

**L-образные линии:** Можно разбить на 2 отрезка (горизонтальный + вертикальный), если хочешь строго как на референсе.

---

## Часть 4: Геометрия и материалы

**Three.js в 2D-режиме:**
- `OrthographicCamera` — без перспективы, «плоский» вид
- Всё лежит в плоскости `z = 0`
- Камера смотрит сверху (`position.z = 5`)

**Геометрии:**
- `CircleGeometry(radius, segments)` — круг
- `PlaneGeometry(w, h)` — квадрат/прямоугольник
- `BufferGeometry` — кастомная геометрия (для линий — 2 треугольника)

**Материал:** `MeshBasicMaterial` — цвет без освещения. Идеально для плоского арта.

---

## Часть 5: Анимация

```js
function animate() {
  requestAnimationFrame(animate);
  time += 0.01;
  // Меняем scale, position, rotation...
  renderer.render(scene, camera);
}
```

**Что можно анимировать:**
- `mesh.scale.setScalar(s)` — пульсация
- `mesh.position.x += 0.001` — дрейф
- `mesh.rotation.z += 0.01` — вращение
- Цвет через `material.color.setHex(...)`

---

## Что менять, чтобы экспериментировать

| Параметр | Где | Эффект |
|----------|-----|--------|
| `bounds` | createNodes | Размер «поля» |
| `density < 0.35` | createNodes | Больше/меньше пустот |
| `maxDist` | createConnections | Длина связей |
| `PALETTE` | сверху | Цветовая гамма |
| `octaves` | fbm | Детализация шума |
| `seed` | createNodes | Другая картина при том же seed |

## Дальше

- **Simplex noise:** Замени `noise2D` на библиотеку `simplex-noise` для более качественного шума
- **Сегментированные круги:** Раздели круг на сектора (как pie chart) — нужна кастомная геометрия или шейдер
- **Шейдеры:** Для тысяч объектов эффективнее один `ShaderMaterial` с procedural паттернами в fragment shader
